Endpoints

GET /api/v1/convert?url={music_url}&targets={list}

Haupt-Endpoint. url Pflicht, targets optional (Default: alle verfügbaren).

GET /api/v1/services

Liefert unterstützte Dienste + Fähigkeiten (welche Content-Typen der Dienst kann).

GET /api/v1/health

Liveness/Readiness: Redis-Verbindung, Adapter-Status. Für Kubernetes Probes.

GET /api/docs

Automatische OpenAPI/Swagger-UI von FastAPI.

Auth & Rate-Limit

Ohne Key: 20 req/min pro IP. Mit X-API-Key-Header: 300 req/min. Limits per Values überschreibbar.

Key-Verwaltung: CLI auf dem Backend (linkhop-admin issue-key --note "paul"). Keys in Redis, nicht in der Seite anfragbar — bewusst low-profile.

Fehlerformat

{
  "error": {
    "code": "unsupported_service",
    "message": "URL host does not match any known service"
  }
}
