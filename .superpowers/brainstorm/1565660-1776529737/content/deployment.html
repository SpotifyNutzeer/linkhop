linkhop — Deployment & Helm-Chart (Abschnitt 5/5)

Generisches Chart, getestet gegen RKE2 + Traefik + cert-manager + external-dns

Chart-Struktur

deploy/helm/linkhop/
├── Chart.yaml
├── values.yaml           # Defaults + kommentiert
├── values.schema.json    # Validierung
└── templates/
    ├── _helpers.tpl
    ├── backend-deployment.yaml
    ├── backend-service.yaml
    ├── frontend-deployment.yaml
    ├── frontend-service.yaml
    ├── ingress.yaml              # traefik/nginx, pathType: Prefix
    ├── redis-*.yaml              # Bitnami-Subchart ODER external
    ├── postgres-connection.yaml  # nur Secret-Ref; DB wird extern erwartet
    ├── configmap.yaml
    ├── secret-refs.yaml          # nur Referenzen, erzeugt keine Creds
    ├── serviceaccount.yaml
    ├── networkpolicy.yaml
    └── poddisruptionbudget.yaml

Values-Highlights

global:
  domain: linkhop.paul.wtf

backend:
  replicas: 2
  image: { repository: ghcr.io/<user>/linkhop-backend, tag: "" }
  resources: { requests: { cpu: 50m, memory: 128Mi }, limits: { cpu: 500m, memory: 512Mi } }
  rateLimit: { anonymous: 20, withKey: 300 }
  cacheTtlSeconds: 604800
  adapters:
    spotify:  { enabled: true, existingSecret: linkhop-spotify }
    deezer:   { enabled: true }  # keine Creds nötig
    tidal:    { enabled: true, existingSecret: linkhop-tidal }
    youtubeMusic: { enabled: true, existingSecret: linkhop-ytm }

frontend:
  replicas: 2
  image: { repository: ghcr.io/<user>/linkhop-frontend, tag: "" }

redis:                # Bitnami-Subchart, kann via external:true deaktiviert werden
  enabled: true
  architecture: standalone
  auth: { existingSecret: linkhop-redis }

postgres:             # extern erwartet, z.B. via cloudnative-pg Cluster
  existingSecret: linkhop-pg      # keys: host, port, user, password, database

ingress:
  enabled: true
  className: traefik
  annotations:
    external-dns.alpha.kubernetes.io/hostname: linkhop.paul.wtf
  tls:
    enabled: true
    issuerRef: letsencrypt-prod  # erzeugt Certificate-CR wenn gesetzt

networkPolicy: { enabled: true }
podDisruptionBudget: { enabled: true, minAvailable: 1 }

Images

Zwei Images: linkhop-backend, linkhop-frontend. Build in GitHub Actions, Push zu ghcr.io. Tags: Git-SHA + SemVer.

Secrets

Chart erzeugt keine API-Credentials. Nur existingSecret-Referenzen. Kompatibel mit external-secrets.

Datenbank

Postgres extern erwartet (cloudnative-pg Cluster separat). Backend läuft Migrationen per alembic als initContainer. Tabellen: conversions, api_keys.

Probes

Backend /api/v1/health. Frontend statisches /. Readiness blockiert Traffic bis Redis+Postgres erreichbar.

Publish

Chart wird in gh-pages (oder ChartMuseum) veröffentlicht. Dann per FluxCD-HelmRelease gegen dein paulwtf-Cluster gefahren, Values als ConfigMap in fluxcd/apps/linkhop/. Ein separater PR in deinem Flux-Repo pinnt die Version.
